# Contributing

Thank you for helping improve this public portfolio.

## What to contribute

- Documentation corrections and clarity improvements
- Broken link fixes
- ADR proposals for site architecture
- Blog drafts aligned with [PUBLIC_PROOF_GATE.md](./PUBLIC_PROOF_GATE.md)
- Accessibility and SEO improvements

## Before opening a pull request

1. Open or reference an issue describing the problem.
2. Keep the change narrowly scoped.
3. Do not include private, client, employer, or proprietary information.
4. Do not claim performance, compliance, readiness, or deployment status without evidence.
5. Run `npm run build` locally when changing docs or site config.

## Pull request standard

Include:

- **Problem** — what was wrong or missing
- **Change** — what you changed
- **Verification** — build output, screenshots if visual
- **Public Proof Gate** — confirm no boundary violations
- **Risks and limitations** — what you intentionally did not change

Use the [pull request template](.github/PULL_REQUEST_TEMPLATE.md).

## Code of conduct

Be respectful and precise. Disagreement is welcome; harassment is not.
