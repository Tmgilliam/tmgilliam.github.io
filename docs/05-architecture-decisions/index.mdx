# Architecture Decisions

Architecture Decision Records (ADRs) capture **significant choices**, their context, and their consequences—so recruiters, collaborators, and future-you can understand _why_, not just _what_.

## ADR index

| ADR                                        | Title                            | Status   |
| ------------------------------------------ | -------------------------------- | -------- |
| [ADR-001](./adr-001-portfolio-platform)    | Portfolio Platform (Docusaurus)  | Accepted |
| [ADR-002](./adr-002-public-proof-boundary) | Public Proof Boundary            | Accepted |
| [ADR-003](./adr-003-static-deployment)     | Static Deployment (GitHub Pages) | Accepted |

## Format

Each ADR includes:

- **Status** — Proposed, Accepted, Deprecated, Superseded
- **Context** — Forces and constraints
- **Decision** — What was chosen
- **Consequences** — Tradeoffs and follow-ups

## Scope

These ADRs govern **this public documentation site and portfolio publishing model**. They do not replace architecture decisions inside private operating systems or client engagements.
