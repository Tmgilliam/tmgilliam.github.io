## Problem

Technical leadership work is hard to evaluate without a durable, recruiter-facing proof surface that separates verified public artifacts from private systems.

## Method

Docusaurus + TypeScript portfolio with GitHub Pages, CI/CD, CodeQL, dependency controls, accessibility posture, curated project catalog, and explicit public/private boundaries.

## What judges can verify

- Live production site
- Public repository and release
- Architecture documentation and ADRs
- Honest project maturity labels
- Sanitized private-system disclosure (BBIOS architecture summary only)
